Ten principles every developer should live by — a guide to building software that is simple, human, and built to last.

The Principles

The Software Manifesto distills decades of hard-won engineering wisdom into ten actionable truths.

  1. Principle 01

    Simplicity Over Cleverness

    Write code that a junior developer can understand at 2 AM. The smartest solution is usually the simplest one — resist the urge to be clever.

  2. Principle 02

    Clarity Is Kindness

    Code is read far more than it is written. Name things honestly, comment why (not what), and treat every future reader as someone you respect.

  3. Principle 03

    Test What You Ship

    Untested code is a liability. Write tests that describe behavior, not implementation — they are the executable documentation of your intentions.

  4. Principle 04

    Own Your Decisions

    Every architectural choice has consequences. Make them deliberately, document the reasoning, and take responsibility when they need to change.

  5. Principle 05

    Collaborate Relentlessly

    Great software is not built in isolation. Share early, review openly, and treat every code review as a learning opportunity rather than a judgment.

  6. Principle 06

    Iterate, Don't Procrastinate

    Shipped and imperfect beats perfect and theoretical. Deliver value early, gather feedback, and improve continuously in small, safe steps.

  7. Principle 07

    Security Is Not Optional

    Security is a feature, not a phase. Build it in from day one — validate inputs, least-privilege by default, and threat-model before you code.

  8. Principle 08

    Performance Is Respect

    Slow software disrespects users' time. Measure before optimizing, optimize where it matters most, and make speed a first-class product requirement.

  9. Principle 09

    Document to Empower

    Documentation is infrastructure. Write it for your future self, for your team, and for the developer who will maintain this system long after you're gone.

  10. Principle 10

    Serve the Human

    Technology exists to solve human problems. Never lose sight of the person on the other side of the screen — their needs are the north star of every line you write.

"Any fool can write code that a computer can understand. Good programmers write code that humans can understand." — Martin Fowler
